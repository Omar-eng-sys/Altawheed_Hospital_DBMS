## 🚀 How to Work on This Project (Git Workflow)

If this is your first time using Git, don't worry! Just open your terminal (Command Prompt or Git Bash) and follow these exact steps:

### Step 1: Download the Project (Do this ONLY ONCE)
To get a copy of the database file on your personal computer, run:
```bash
git clone [https://github.com/Omar-eng-sys/Altawheed_Hospital_DBMS.git](https://github.com/Omar-eng-sys/Altawheed_Hospital_DBMS.git)
```

### Step 2: Get the Latest Updates (Do this EVERY TIME you start working)
Before you write any code, make sure you have the latest updates from the team to avoid conflicts:
```bash
cd Altawheed_Hospital_DBMS
git pull
```

### Step 3: Write Your Code (The SQL Rules)
Open the `Hospital_Management_System.sql` file in any text editor (like VS Code or Notepad).
**⚠️ STRICT DEVELOPMENT RULES:**
1. **DO NOT** write code directly into the shared file from scratch.
2. Open your Oracle SQL Command Line (`SQL*Plus`).
3. Write your `CREATE TABLE`, `ALTER`, or `INSERT` commands there and test them.
4. Once the terminal says **`Table created`** or **`Row created`** successfully, **COPY** that exact code and **PASTE** it into your specific `TODO` section in the shared `.sql` file.

### Step 4: Save and Upload Your Work (Push)
Once you have pasted your successful code into the file and saved it, send your updates back to GitHub by running these 3 commands in order:

1. Tell Git to track your changes:
```bash
git add .
```
2. Save the changes with a short, clear message explaining what you did (e.g., "Created Patient and Dependent tables"):
```bash
git commit -m "Your clear message here"
```
3. Upload the changes to GitHub so the rest of the team can see them:
```bash
git push
```
